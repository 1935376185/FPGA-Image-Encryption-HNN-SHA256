## Overview

This project implements a **hardware accelerated image encryption/decryption system** based on a **Hopfield Neural Network (HNN)** and **SHA-256** hash function, targeting the **Xilinx Zynq-7020** SoC (ZedBoard development board). 

The design is developed in **Vivado 2024.2** and can be directly deployed on any board with a Zynq-7020 or compatible 7‑series FPGA. It is recommended to use Vivado **2024.2 or later** to open the `.xpr` project file.

## Requirements

- **Vivado**: 2024.2 or higher (2024.2, 2025.1, etc.)
- **Hardware**: Zynq-7020 (XC7Z020CLG484-2) or similar 7‑series device
- **Memory**: 512 MB DDR3 (on‑board)

- **Board**: Xiaomeige ZYNQ7020 (XC7Z020), tested on Vivado 2024.2.

### VIO Control Signals

The following VIO (Virtual Input/Output) probes are used to control the core functionality. All signals are active-high logic unless stated otherwise.

| Signal       | Bit Width | Description                                                  |
| ------------ | --------- | ------------------------------------------------------------ |
| `denc`       | 1         | **Read/Write direction**.  - `0`: Read from **Left** image buffer, write to **Right** image buffer.  - `1`: Read from **Right** image buffer, write to **Left** image buffer. |
| `caseTest`   | 2         | **Operation mode** (4 selections).  - `00 (0)`: **Copy & Paste** — straight pixel copy without any transformation.  - `01 (1)`: **Encryption/Decryption & Paste** — apply encryption algorithm and paste the result.  - `10 (2)`: **Encryption + Gaussian Noise** — encrypt then add Gaussian noise with variance **0.1**.  - `11 (3)`: **Encryption + Salt & Pepper Noise** — encrypt then add salt & pepper noise with density **0.4**. |
| `soft_reset` | 1         | **System reset / start**.  - `0`: **Reset** — all internal state machines and FIFOs are held in reset.  - `1`: **Start** — release reset and begin processing. |

> **Note**
>
> - The `soft_reset` signal must be set to `0` (reset) before changing `denc` or `caseTest` to avoid undefined behavior.
> - After new parameters are set, assert `soft_reset = 1` to start the operation.
> - The encryption algorithm used in `caseTest = 1/2/3` is a lightweight XOR‑based cipher (see section [Encryption Core](#encryption-core) for details).
> - Gaussian noise (σ = 0.1) and salt & pepper noise (density = 0.4) are generated by an on‑chip LFSR‑based PRNG.

### Example Sequence

1. Set `soft_reset = 0` (reset).
2. Set `denc = 0` (read left, write right) and `caseTest = 1` (encrypt/decrypt & paste).
3. Set `soft_reset = 1` (start) — the system will read from the left buffer, encrypt the data, and write the ciphertext to the right buffer.
4. Once done, set `soft_reset = 0` again, switch `denc = 1` and `caseTest = 3` to decrypt and add salt & pepper noise, then assert `soft_reset = 1` to execute the reverse operation.

This procedure can be monitored via the VIO waveform as shown in **Figure 1**.
